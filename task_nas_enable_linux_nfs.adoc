= Habilitar armazenamento NAS para servidores Linux usando NFS com o ONTAP System Manager
:hardbreaks:
:toclevels: 1
:allow-uri-read: 
:toclevels: 1
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ./media/


[role="lead"]
Crie ou modifique VMs de storage para habilitar servidores NFS para fornecer dados a clientes Linux.

Ative uma VM de storage nova ou existente para o protocolo NFS usando este procedimento.

image:workflow_nas_enable_linux_nfs.png["Resumo do fluxo de trabalho: 1 Ativar a política de exportação vol de raiz aberta do NFS 2 3 Configurar DNS 4 Configurar serviços de nomes 5 Configurar a encriptação, se necessário"]

.Antes de começar
Certifique-se de que anotou os detalhes de configuração de qualquer rede, autenticação ou serviços de segurança necessários no seu ambiente.

.Passos
. Habilite o NFS em uma VM de storage.
+
** Para novas VMs de armazenamento: selecione *Armazenamento > VMs de armazenamento*, selecione *Adicionar*, insira um nome para a VM de armazenamento e, na guia *SMB/CIFS, NFS, S3*, selecione *Habilitar NFS*.
+
... Confirme o idioma predefinido.
... Adicione interfaces de rede.
... Atualizar as informações da conta do administrador da VM de armazenamento (opcional).


** Para VMs de armazenamento existentes: selecione *Cluster > VMs de armazenamento*, selecione a VM de armazenamento, selecione *Configurações* e, em seguida, selecione image:icon_gear.gif["Ícone ações"] ou image:icon-edit-pencil-blue-outline.png["Ícone de edição"] em *NFS* e selecione *Habilitar NFS*.
+
... Ative ou desative versões e opções específicas do protocolo NFS conforme necessário.
+

NOTE: O monitoramento de eventos de acesso a arquivos NFSv4.2 pelo FPolicy não é suportado. Se você configurar o monitoramento pelo FPolicy, será necessário desativar o NFSv4.2 na SVM. Para referência, consulte link:./nfs-admin/enable-disable-nfs-task.html["Ativar ou desativar NFS para ONTAP SVMs"].





. Abra a política de exportação do volume raiz da VM de storage:
+
.. Selecione *Armazenamento > Volumes*, selecione o volume raiz da VM de armazenamento (que por padrão é _volume-name_ _root), e, em seguida, selecione a política exibida em *Export Policy*.
.. Selecione *Adicionar* para adicionar uma regra.
+
*** Especificação do cliente `0.0.0.0/0`
*** Protocolos de acesso: NFS
*** Detalhes de acesso: UNIX Read-only




. Configure DNS para resolução de nomes de host: selecione *Armazenamento > VMs de armazenamento*, selecione a VM de armazenamento, selecione *Configurações* e, em seguida, selecione image:icon_gear.gif["Ícone ações"] em *DNS*.
. Configure os serviços de nomes conforme necessário.
+
.. Selecione *Armazenamento > VMs de armazenamento*, selecione a VM de armazenamento, selecione *Configurações* e, em seguida, selecione image:icon_gear.gif["Ícone ações"] LDAP ou NIS.
.. Selecione image:icon_pencil.gif["Ícone de edição"] no bloco Name Services Switch para incluir quaisquer alterações.


. Configure Kerberos se necessário:
+
.. Selecione *Armazenamento > VMs de armazenamento*, selecione a VM de armazenamento e, em seguida, selecione *Configurações*.
.. Selecione image:icon_arrow.gif["Ícone de seta"] na caixa Kerberos e, em seguida, selecione *Adicionar*.
