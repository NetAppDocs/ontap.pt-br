= Saiba mais sobre como gerenciar snapshots locais do ONTAP
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Um _snapshot_ é uma imagem pontual e somente leitura de um volume. A imagem consome espaço de armazenamento mínimo e incorre em sobrecarga de desempenho insignificante, pois Registra apenas alterações nos arquivos desde o último snapshot.

Você pode usar um snapshot para restaurar todo o conteúdo de um volume ou para recuperar arquivos individuais ou LUNs. Os snapshots são armazenados no diretório `.snapshot` no volume.

No ONTAP 9.4 e posteriores, um FlexVol volume pode conter até 1.023 instantâneos.  No ONTAP 9.3 e versões anteriores, um volume pode conter até 255 instantâneos.

[NOTE]
====
A partir do ONTAP 9,8, os volumes FlexGroup podem conter 1023 snapshots. Para obter mais informações, link:../flexgroup/protect-snapshot-copies-task.html["Proteger volumes FlexGroup usando snapshots"]consulte .

====
Existem vários tipos comuns de snapshots do ONTAP:

* `sched_created`: Instantâneos gerados automaticamente por uma política de instantâneos do ONTAP, com base em programações predefinidas (por exemplo, horárias, diárias, semanais ou mensais).
* `sm_created`: Instantâneos criados automaticamente como resultado de transferências de replicação do SnapMirror (operações de atualização ou inicialização).
* `application_created`: Snapshots gerados por software de integração em nível de host ou em nível de banco de dados, como SnapCenter ou SnapDrive.
* `user_created`: Snapshots acionados manualmente por um administrador usando a ONTAP CLI ou System Manager.
* `sv_created`: Instantâneos criados automaticamente durante operações de cluster, como DR de Storage Virtual Machine (SVM) ou fluxos de trabalho de proteção específicos.
* `luns_created`: Instantâneos criados para fazer backup ou capturar o estado de clones de LUN ou namespace NVMe.
