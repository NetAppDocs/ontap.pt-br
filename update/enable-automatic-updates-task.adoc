= Ativar atualizações automáticas
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Utilize System Manager para ativar as atualizações automáticas e permitir que ONTAP baixe e instale firmware, sistema e atualizações de segurança sem a sua intervenção.

Você pode usar a interface de linha de comando do ONTAP para gerenciar apenas as atualizações de firmware do SP/BMC.

* Saiba mais sobre link:../system-admin/methods-manage-sp-bmc-firmware-updates-concept.html["Gerenciando atualizações de firmware do ONTAP SP/BMC"].
* Saiba mais sobre as diferenças entre System Manager e a interface de linha de comando no gerenciamento de atualizações automáticas no artigo da Knowledge Base: link:https://kb.netapp.com/on-prem/ontap/Ontap_OS/OS-KBs/Why_do_GUI_and_CLI_show_different_status_for_ONTAP_automatic_updates_configuration["Por que a GUI e a CLI mostram status diferentes para a configuração de atualizações automáticas do ONTAP?"].


A disponibilidade de atualizações automáticas para cada componente do sistema depende da sua versão do ONTAP .

[cols="25,75"]
|===
| Versão de ONTAP | Atualizações automáticas disponíveis 


| 9.16.1 e mais tarde  a| 
* Proteção autônoma contra ransomware com inteligência artificial (ARP/AI)
* Base de dados de Fuso horário do ONTAP
* Firmware de storage para dispositivos de storage, discos e compartimentos de disco
* Firmware SP/BMC para processadores de serviço e módulos BMC
* Pacote de Qualificação de disco (DQP)




| 9.13.1 e mais tarde  a| 
* Base de dados de Fuso horário do ONTAP
* Firmware de storage para dispositivos de storage, discos e compartimentos de disco
* Firmware SP/BMC para processadores de serviço e módulos BMC
* Pacote de Qualificação de disco (DQP)




| 9.10.1 e mais tarde  a| 
* Firmware de storage para dispositivos de storage, discos e compartimentos de disco
* Firmware SP/BMC para processadores de serviço e módulos BMC
* Pacote de Qualificação de disco (DQP)


|===
Se uma atualização automática não estiver disponível para a sua versão do ONTAP ou se não tiver atualizações automáticas ativadas, pode executar atualizações de firmware, base de dados de fuso horário e segurança manualmente.

.Sobre esta tarefa
Para habilitar atualizações automáticas, é necessário aceitar os termos de um EULA e confirmar ou configurar o comportamento de atualização padrão para cada componente do sistema, conforme apropriado para seu ambiente.

Para clusters que foram criados no ONTAP 9.16.1 ou anterior e depois atualizados para 9.16.1 ou posterior:

* Se a funcionalidade de atualizações automáticas estiver ativada antes da atualização, os padrões de atualização dos componentes do sistema não serão alterados após a atualização.
* Se as atualizações automáticas não estiverem ativadas antes da atualização, os padrões de atualização dos componentes do sistema no ONTAP 9.16.1 e versões posteriores serão aplicados.


Os padrões de atualização automática de componentes do sistema são definidos para atualizar automaticamente ou mostrar notificações, dependendo da sua versão do ONTAP .  Confirme se essas configurações estão corretas para seu ambiente antes de concluir o procedimento.

 https://www.youtube.com/watch?v=GoABILT85hQ["vídeo"^]Isso mostra uma visão geral rápida do uso do processo de atualização automática.

.Antes de começar
* Você precisa ter um direito de suporte atual. Você pode verificar isso na link:https://mysupport.netapp.com/site/["Site de suporte da NetApp"^]página *Detalhes do sistema*.
* Para ativar as atualizações automáticas, você deve primeiro ativar o AutoSupport com HTTPS. Se o AutoSupport não estiver ativado no cluster ou se o AutoSupport estiver ativado no cluster com outro protocolo de transporte, pode ativá-lo com HTTPS durante este procedimento.



NOTE: O AutoSupport OnDemand é ativado por padrão e funcional quando configurado para enviar mensagens para suporte técnico usando o protocolo de transporte HTTPS.

* A partir do ONTAP 9.10.1, se você habilitar atualizações automáticas, certifique-se de ter conetividade HTTPS com os seguintes URLs adicionais:
+
** https://support-sg-naeast.NetApp.com
** https://support-sg-nawest.NetApp.com




.Passos
[role="tabbed-block"]
====
.Gestor do sistema - ONTAP 9.16.1 e posterior
--
. No System Manager, selecione *Cluster > Settings*.
. Em *Atualizações de software*, selecione *Ativar*.
. Em *Configurações de atualização automática*, selecione a ação a ser executada para cada tipo de atualização.
+
*Atualizar automaticamente* está selecionada por padrão para todas as opções. Você pode escolher modificar a seleção padrão para *Mostrar notificações* ou *Descartar as atualizações automaticamente*.

. Selecione *Aceito para a atualização atual e todas as futuras* e depois selecione *Salvar*.


--
.Gestor do sistema - ONTAP 9.15.1 e anterior
--
. No System Manager, selecione *Cluster > Settings*.
. Em *Atualização automática*, selecione image:icon_kabob.gif["seleção de menu"] e depois selecione *Ativar*.
+
*Atualizar automaticamente* está selecionada por padrão para todas as opções. Você pode escolher modificar a seleção padrão para *Mostrar notificações* ou *Descartar as atualizações automaticamente*.

. Selecione *Aceito esta atualização e todas as futuras* e depois selecione *Salvar*.


--
.CLI
--
Você pode usar a interface de linha de comando do ONTAP para gerenciar atualizações de firmware do SP/BMC somente

. Ativar atualizações automáticas de firmware:
+
[source, cli]
----
system service-processor image modify -node <node_name> -autoupdate true
----
