= SnapMirror backups na nuvem para storage de objetos
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
_SnapMirror Cloud_ é uma tecnologia de backup e recuperação projetada para usuários do ONTAP que desejam migrar seus fluxos de trabalho de proteção de dados para a nuvem ou para uma solução de storage de objetos local. Organizações que estão abandonando arquiteturas legadas de backup em fita podem usar storage de objetos como repositório alternativo para retenção de dados e arquivamento a longo prazo. SnapMirror Cloud oferece replicação de ONTAP para storage de objetos como parte de uma estratégia de backup incremental contínuo.

SnapMirror Cloud é um recurso licenciado do ONTAP. SnapMirror Cloud foi introduzido no ONTAP 9.8 como uma extensão da família de tecnologias de replicação SnapMirror. Embora SnapMirror seja frequentemente usado para backups ONTAP-para-ONTAP, SnapMirror Cloud utiliza o mesmo mecanismo de replicação para transferir snapshots do ONTAP para backups em storage de objetos compatível com S3.

Projetado para casos de uso de backup, o SnapMirror Cloud oferece suporte tanto à retenção de longo prazo quanto a fluxos de trabalho de arquivamento. Assim como no SnapMirror, o backup inicial do SnapMirror Cloud realiza uma transferência completa de um volume. Para backups subsequentes, o SnapMirror Cloud gera um snapshot do volume de origem e transfere o snapshot, contendo apenas os blocos de dados alterados, para um storage de objetos de destino.

SnapMirror Cloud pode ser configurado entre sistemas ONTAP e determinados storage de objetos locais e de nuvem pública, incluindo Amazon S3, Google Cloud Storage e Microsoft Azure Blob Storage. Alvos adicionais de storage de objetos locais incluem StorageGRID e ONTAP S3.

Além de usar ONTAP System Manager para gerenciar as configurações do SnapMirror Cloud, várias opções de orquestração estão disponíveis para gerenciar os backups do SnapMirror Cloud:

* Diversos parceiros de backup terceirizados oferecem suporte para a replicação em nuvem do SnapMirror. Os fornecedores participantes estão disponíveis no link:https://www.netapp.com/blog/new-backup-architecture-snapdiff-v3/["NetApp blog"^].
* NetApp Backup and Recovery para uma solução nativa NetApp para ambientes ONTAP
* APIs para desenvolver software personalizado para workflows de proteção de dados ou aproveitar ferramentas de automação



NOTE: SnapMirror Cloud, a replicação gerenciada pelo System Manager, suporta apenas StorageGrid e ONTAP S3 como destino de storage de objetos. Outros provedores não são suportados.


NOTE: O uso das APIs de SnapMirror Cloud geralmente é restrito a parceiros de backup. Clientes que desejam desenvolver suas próprias soluções baseadas em API devem entrar em contato com sua equipe de contas da NetApp para obter informações sobre os termos da licença.
