= Crie um snapshot comum entre grupos de consistência do ONTAP ou SVMs NAS
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
Além das operações de snapshot agendadas regularmente, você pode criar manualmente um link:../concepts/snapshot-copies-concept.html["snapshot"] comum entre os volumes no grupo de consistência SnapMirror primário e os volumes no grupo de consistência SnapMirror secundário em uma configuração SAN ou no nível da SVM para configurações NAS.

.Sobre esta tarefa
O intervalo de criação de snapshots agendado é de 12 horas para a política de failover automatizado e de 3 horas para a política de failover duplex automatizado.

.Antes de começar
* A relação de grupo SnapMirror deve estar sincronizada.


.Passos
Você pode criar um snapshot comum usando System Manager ou a CLI do ONTAP.

[role="tabbed-block"]
====
.System Manager
--
. No cluster de destino, navegue até *Proteção > Replicação*.
. Localize a fonte do relacionamento, selecione image:icon_kabob.gif["Ícone de opções do menu"] e escolha *Atualizar*.
. Escolha *Conforme a política* ou *Selecione o instantâneo* para escolher um instantâneo existente.
. Selecione *Atualizar*.


--
.CLI
--
. A partir do site de destino, crie um instantâneo comum:
+
`snapmirror update -destination-path <destination path>`

+
Exemplo de configuração SAN:

+
[listing]
----
destination::>snapmirror update -destination-path vs1_dst:/cg/cg_dst
----
+
Exemplo de configuração de NAS:

+
[listing]
----
snapmirror update -destination-path vs_b:
----
. Monitorize o progresso da atualização:
+
`destination::>snapmirror show -fields newest-snapshot`
