= Instalar ou atualizar o ONTAP Mediator
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Para instalar ou atualizar o ONTAP Mediator, você precisa atender a todos os pré-requisitos, baixar o pacote de instalação e executar o instalador no host.

* A partir do ONTAP 9.8, você pode usar qualquer versão do ONTAP Mediator para monitorar uma relação de sincronização ativa do SnapMirror.
* Ao configurar o ONTAP Mediator com o protocolo iSCSI, você pode usar qualquer versão do ONTAP Mediator para monitorar uma configuração IP do MetroCluster.
* A partir do ONTAP 9.19.1 e ONTAP Mediator 1.12, você pode configurar um ONTAP Mediator que usa HTTPS para comunicação para monitorar sua configuração de IP do MetroCluster.




== Considerações sobre instalação e atualização

Analise as seguintes informações antes de atualizar ou instalar ONTAP Mediator.


NOTE: O ONTAP Mediator 1.8 e versões anteriores não são compatíveis com o modo FIPS do Red Hat Enterprise Linux (RHEL), o que impede sua instalação bem-sucedida.  Você pode verificar se o modo FIPS está ativado usando o `fips-mode-setup --check` comando.  Você pode desativar o modo FIPS usando o `fips-modesetup --disable` comando. Reinicie após desabilitar o modo FIPS para instalar com sucesso o ONTAP Mediator 1.8 ou anterior.

* Você deve atualizar o ONTAP Mediator para a versão mais recente.  As versões mais antigas ainda funcionam com todas as versões do ONTAP , mas as versões mais recentes incluem correções de segurança para componentes de terceiros.
* Ao atualizar para uma nova versão do ONTAP Mediator que inclui o pacote SCST, o instalador atualiza automaticamente para a versão recomendada do SCST, a menos que uma versão superior esteja disponível. Para instruções sobre como instalar manualmente uma versão superior do SCST, consulte link:manage-task.html["Gerenciar o ONTAP Mediator"]. Para versões compatíveis, consulte o link:whats-new-concept.html#scst-support-matrix["Matriz de suporte SCST"].


[NOTE]
====
* Se a instalação falhar, talvez seja necessário atualizar para uma versão mais recente do ONTAP Mediator.
* A partir de 15 de junho de 2025, você não poderá instalar ou atualizar o ONTAP Mediator 1.9 e 1.8 porque seus certificados de assinatura de código expiraram.  Se a instalação ou atualização falhar, use a versão de patch do ONTAP Mediator 1.9.1.


====
* Se você instalar o `yum-utils` pacote, você pode usar o `needs-restarting` comando.
* A partir da versão 1.11 do ONTAP Mediator, o IPv6 é suportado para configurações de IP do MetroCluster .




=== Opções de instalação HTTPS e iSCSI para ONTAP Mediator

* No ONTAP Mediator, o pacote SCST é usado para fornecer iSCSI para comunicação em MetroCluster IP configurações. O pacote SCST é um módulo do kernel compilado durante a instalação. A partir do ONTAP Mediator 1.12, é possível configurar um ONTAP Mediator que utiliza HTTPS em vez de iSCSI para monitorar uma MetroCluster IP configuração.


[CAUTION]
====
* Quando o ONTAP Mediator é configurado com o protocolo HTTPS nas configurações de IP do MetroCluster, uma única instância do ONTAP Mediator só pode monitorar uma configuração de IP do MetroCluster. O monitoramento simultâneo de múltiplas configurações de IP do MetroCluster não é suportado.
* Se você precisar do ONTAP Mediator configurado com HTTPS para monitorar várias configurações de IP do MetroCluster, deverá configurar instâncias separadas do ONTAP Mediator para cada configuração de IP do MetroCluster. Você deve avaliar cuidadosamente os requisitos do seu ambiente de IP do MetroCluster antes de optar por configurar o ONTAP Mediator com HTTPS para MetroCluster IP.


====
* Ao instalar ou atualizar para ONTAP Mediator 1.12 ou posterior, você tem a opção de instalar o ONTAP Mediator com suporte a iSCSI e HTTPS, ou somente com suporte a HTTPS. Se optar por instalar o ONTAP Mediator apenas com suporte a HTTPS, o pacote SCST não será instalado.
* Se você atualizar do ONTAP Mediator 1.11 ou anterior com o SCST instalado e escolher a opção somente HTTPS durante a atualização, o pacote SCST será desinstalado e a nova versão do ONTAP Mediator não conterá o pacote SCST.
* A instalação do ONTAP Mediator com suporte apenas para HTTPS é compatível com os seguintes casos de uso:
+
** Você planeja usar apenas o ONTAP Mediator para monitorar seus clusters de sincronização ativa do SnapMirror.
** Você planeja configurar ONTAP Mediator com HTTPS para monitorar uma única configuração de IP do MetroCluster por instância do ONTAP Mediator.


* ONTAP Mediator deve ser instalado com suporte para iSCSI e HTTPS no seguinte cenário:
+
** Você planeja configurar ONTAP Mediator com iSCSI para monitorar uma ou mais configurações de IP do MetroCluster.

== Requisitos de firewall e rede

ONTAP Mediator utiliza diversas portas para se comunicar com serviços específicos.

* Se você estiver usando um firewall de terceiro:
+
** O acesso HTTPS deve estar ativado.
** Deve ser configurado para permitir o acesso nas portas 31784 e 3260.
+
Ao usar o firewall padrão do Red Hat ou CentOS, o firewall é configurado automaticamente durante a instalação do Mediator.



* Para hosts Linux sem acesso à Internet, certifique-se de que os pacotes necessários estejam disponíveis em um repositório local.
+
Se você estiver usando o Link Aggregation Control Protocol (LACP) em um ambiente Linux, configure o kernel e defina `sysctl net.ipv4.conf.all.arp_ignore` como `2`.

=== Requisitos de atualização do sistema operacional e compatibilidade do kernel

Os seguintes requisitos se aplicam quando o pacote SCST é instalado com ONTAP Mediator.


NOTE: Se você instalou ONTAP Mediator 1.12 ou posterior com suporte apenas para HTTPS, o pacote SCST não está instalado, portanto os requisitos de compatibilidade do kernel não se aplicam.

* Você pode atualizar todos os pacotes de biblioteca, exceto o kernel, mas talvez seja necessário reiniciar o sistema para aplicar as alterações no ONTAP Mediator.  Programe um período de inatividade caso precise reiniciar.
* Você deve manter o kernel do sistema operacional atualizado.  Atualize o núcleo do kernel para uma versão compatível listada emlink:whats-new-concept.html#scst-support-matrix["Matriz de versão do Mediador ONTAP"] .  Você precisa reiniciar o sistema, portanto, planeje uma janela de manutenção para a interrupção.
+
** Desinstale o módulo do kernel SCST antes de reiniciar e reinstale-o em seguida.
** Prepare uma versão compatível do SCST para reinstalar antes de iniciar a atualização do kernel do sistema operacional.




[NOTE]
====
* A versão do kernel deve corresponder à versão do sistema operacional.
* Não atualize o kernel para uma versão do sistema operacional superior à suportada pela sua versão do ONTAP Mediator, pois o módulo SCST testado pode não funcionar.


====


== Instale o ONTAP Mediator quando a Inicialização segura UEFI estiver ativada

O ONTAP Mediator pode ser instalado em um sistema com ou sem a Inicialização segura UEFI ativada.

.Sobre esta tarefa
* Você pode optar por desativar a Inicialização segura UEFI antes de instalar o ONTAP Mediator se não for necessário ou se você estiver solucionando problemas de instalação do ONTAP Mediator. Desative a opção UEFI Secure Boot nas configurações da máquina.
* Se você instalou ONTAP Mediator 1.12 ou posterior com suporte apenas para HTTPS, o pacote SCST não está instalado e você pode ignorar esta tarefa.


[NOTE]
====
Para obter instruções detalhadas sobre como desabilitar o UEFI Secure Boot, consulte a documentação do sistema operacional do host.

====
Para instalar o ONTAP Mediator com o UEFI Secure Boot habilitado, você deve registrar uma chave de segurança antes que o serviço possa iniciar. A chave é gerada durante a etapa de compilação da instalação do SCST e salva como um par de chaves público-privado em sua máquina. Use o `mokutil` utilitário para adicionar a chave pública como uma chave de proprietário de máquina (MOK) ao firmware UEFI, permitindo que o sistema confie e carregue o módulo assinado. Salve a `mokutil` senha em um local seguro, pois isso é necessário ao reiniciar seu sistema para ativar o MOK.

.Passos
. [[STEP_1_uefi]]Verifique se o UEFI Secure Boot está ativado no seu sistema:
+
`mokutil --sb-state`

+
Os resultados indicam se o UEFI Secure Boot está ativado neste sistema.
